# 01. 概要 — homenet とは / Web アプリの超基礎

## homenet は何をするもの？

自宅の LAN（家庭内ネットワーク）には、思ったよりたくさんの機器がつながっています。ルーター、NAS、デスクトップPC、ノートPC、スマホ、テレビ、ゲーム機、スマートスピーカー、照明ハブ、カメラ……。

そのうち「あれ、この IP アドレスの機器なんだっけ？」「あの Mac のメモリって何GBだっけ？」が、ふとした時に分からなくなります。homenet は、こうした **機器の情報を1か所にまとめて、ブラウザからすぐ引ける台帳**です。

- 機器ごとに **IP アドレス / MAC アドレス / ホスト名 / スペック（CPU・メモリ・ストレージ）/ 接続方法 / 設置場所 / 購入情報 / メモ** を記録できます。
- 家のネットワーク全体を **地図（トポロジーマップ）** として俯瞰できます。
- 機器を **追加・編集・削除** できます（ブラウザの画面から、またはファイルを直接書き換えて）。

### 何ではないか（やらないこと）

- 複数人で使う共有サービスではありません（**自分1人用**）。
- インターネットに公開しません（**家の中＝LAN 内だけ**）。
- リアルタイム監視ツールでもセキュリティ製品でもありません（CPU使用率などのライブ監視は将来の拡張）。

「**自宅機器の住所録・カタログ**」とイメージすると近いです。

## そもそも「Web アプリ」とは？

homenet は **Web アプリケーション**です。普段使う「アプリ」には2種類あります。

1. **インストール型アプリ**: パソコンやスマホに入れて使う（例: Word、LINE アプリ）。
2. **Web アプリ**: **ブラウザ**（Chrome / Safari / Edge など）で開いて使う（例: Gmail、YouTube、ネットバンキング）。homenet はこちら。

Web アプリは、おおまかに次の2つの部品でできています。これは homenet も同じです。

```
  あなた                  [ フロントエンド ]              [ バックエンド ]
 (ブラウザ)  ── 見る ──►   画面・見た目      ── 頼む ──►   データ保管・処理
                          React / nginx     ◄── 返す ──    FastAPI / devices.json
                          「お店の接客係」                 「お店の厨房・倉庫」
```

- **フロントエンド (frontend)** = あなたが**見て触る画面**。ボタン、地図、一覧表などの見た目。ブラウザの中で動きます。
  - homenet では **React + TypeScript** という道具で作り、**nginx（エンジンエックス）** というソフトが配っています。
- **バックエンド (backend)** = 画面の裏で**データを保管したり計算したりする係**。あなたの目には直接見えません。
  - homenet では **FastAPI**（Python 製）という道具で作っています。データは `devices.json` というファイルに保存します。

フロントエンドとバックエンドは、**API（エーピーアイ）** という「決まった形のやり取り」で会話します。たとえば画面が「機器の一覧をください」とお願いすると、バックエンドが一覧を返す、という具合です。

> 🔰 もっと詳しい仕組みは [03 アーキテクチャ](03-architecture.md) で図を使って説明します。ここでは「**画面係（フロント）と倉庫係（バック）がペアで動く**」とだけ覚えれば十分です。

## どうやって動かすの？（Docker）

homenet は「フロント」と「バック」の2つを動かす必要があります。普通にやると、Python を入れたり、Node.js を入れたり、設定をそろえたり…と準備が大変です。

そこで **Docker（ドッカー）** を使います。Docker は、**アプリと必要な部品を箱（コンテナ）に丸ごと詰めて、どのパソコンでも同じように動かす**仕組みです。

- 例えるなら「電子レンジで温めるだけの冷凍弁当」。中身（材料・調理）は箱に入っているので、あなたは**温める（起動する）だけ**。
- homenet では **2つの箱（frontend と backend）** を、**docker compose** という道具でまとめて起動します。
- あなたが打つのは基本これだけ:

```bash
docker compose up -d --build
```

このあとブラウザで `http://localhost:8080` を開けば、homenet が表示されます。

> 🔰 Docker のインストールから起動までの**具体的な手順**は [02 セットアップと起動](02-getting-started.md) にあります。

## 用語ミニ表（最初の5つだけ）

| 言葉 | ざっくり意味 |
|---|---|
| ブラウザ | Web を見るソフト（Chrome / Safari / Edge など） |
| フロントエンド | 画面・見た目の部分 |
| バックエンド | データを保管・処理する裏方 |
| API | フロントとバックの「決まった会話のしかた」 |
| Docker | アプリを箱に詰めて簡単に動かす仕組み |

> もっとたくさんの用語は [07 用語集](07-glossary.md) にまとめてあります。分からない言葉が出てきたら、そこを引いてください。
