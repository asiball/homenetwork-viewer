# 03. アーキテクチャ — 何がどう動いているか

ここでは homenet の中身を、Web の基礎から順に説明します。コードを書き換えなくても、**全体像を持っておくと運用やトラブル対応がぐっと楽**になります。

## 登場人物（4つ）

| 名前 | 役割 | たとえ |
|---|---|---|
| **ブラウザ** | あなたが操作する。画面を表示し、クリックを送る | お客さん |
| **nginx**（フロント側） | 画面ファイルを配り、データの問い合わせは backend へ取り次ぐ | お店の入口・案内係 |
| **FastAPI**（バック側） | データを読み書きして答える | 厨房・倉庫係 |
| **homenet.db** | 機器データの保管先（SQLite データベース） | 在庫ノート |

---

## 全体図

```
        あなたのPC（同じ家のLAN）
   ┌───────────────────────────────────────────────────────────────┐
   │                                                                 │
   │   ブラウザ                                                       │
   │   http://localhost:8080                                         │
   │      │                                                          │
   │      │  ①画面ちょうだい / ②データちょうだい                      │
   │      ▼                                                          │
   │  ┌───────────────── frontend コンテナ ─────────────────┐        │
   │  │  nginx                                               │        │
   │  │   ・/        → React で作った画面ファイルを返す（①）  │        │
   │  │   ・/api/... → backend へ“取り次ぐ”（リバースプロキシ）│        │
   │  └───────────────────────┬─────────────────────────────┘        │
   │                          │ ②/api/... を中継                      │
   │                          ▼                                      │
   │  ┌───────────────── backend コンテナ ──────────────────┐        │
   │  │  FastAPI (Python)                                    │        │
   │  │   ・GET  /api/devices     一覧を返す                  │        │
   │  │   ・POST /api/devices     追加する                    │        │
   │  │   ・PUT  /api/devices/:id  更新する                   │        │
   │  │   ・DELETE /api/devices/:id 削除する                  │        │
   │  └───────────────────────┬─────────────────────────────┘        │
   │                          │ 読み書き                              │
   │                          ▼                                      │
   │                   data/homenet.db   ← あなたのPC上のSQLite DB     │
   │                   （編集は画面／export→import 経由）              │
   └───────────────────────────────────────────────────────────────┘
```

ポイントは **入口は1つ（8080番の nginx）だけ**ということ。バックエンドは外に出さず、nginx が裏で取り次ぎます。だから「画面」も「データ」も同じ `http://localhost:8080` の住所で扱え、設定がシンプルになります。

## 用語をかみ砕く

### HTTP と「リクエスト / レスポンス」

ブラウザとサーバーは **HTTP** という言葉で会話します。会話は必ず「**お願い（リクエスト）→ 返事（レスポンス）**」のペア。

- 「`GET /api/devices` ください」→ サーバー「はい、機器22件のデータ（JSON）です」
- **GET=取得、POST=新規作成、PUT=置き換え（更新）、DELETE=削除**。郵便でいう「取り寄せ・投函・差し替え・破棄」のイメージです。

### JSON

データのやり取りや保存に使う**文字だけの形式**。人間にも読めます。homenet の機器1件はこんな形:

```json
{
  "id": "nas",
  "name": "NAS",
  "ip": "192.168.1.10",
  "mac": "AA:BB:CC:00:0A:11",
  "group": "Infra",
  "online": true
}
```

`homenet.db` は、この機器データが入った1つの SQLite データベースです（→ [05](05-data-and-customize.md)）。

### フロントエンド（React / SPA）

画面側は **React + TypeScript** で作られた **SPA（Single Page Application）** です。

- **SPA** = 最初に画面一式を読み込んだ後は、ページ全体を再読み込みせず、必要なデータだけ API でやり取りして**画面の一部をすばやく書き換える**作り。地図のノードをクリックすると右側だけ切り替わる、あの動きです。
- 画面の住所は `/`（ホーム）、`/d/nas`（詳細）、`/add`（追加）のように変わりますが、これは**ブラウザ内での切り替え**で、毎回サーバーに取りに行っているわけではありません。

### nginx（リバースプロキシ）

**nginx** は2つの仕事をします。

1. React で作った画面ファイル（HTML/JS/CSS）を配る。
2. `/api/...` への問い合わせを **backend に取り次ぐ**（これを**リバースプロキシ**と呼びます）。

> 🔰 **なぜ取り次ぐ？** ブラウザから backend を直接叩く構成だと「別の住所どうしの通信」になり、ブラウザの安全機能（CORS）でつまずきがち。nginx が窓口を1つにまとめることで、同じ住所の中で完結し、設定が楽で安全になります。

### バックエンド（FastAPI）

**FastAPI** は Python 製の API サーバー。homenet では機器データの **取得・追加・更新・削除** を担当し、保存先は SQLite（`homenet.db`）。書き込みは**トランザクション**で行われ、失敗すれば巻き戻るので、途中で電源が落ちてもデータが半端な状態になりにくくなっています。

### Docker（イメージ / コンテナ / ボリューム）

- **イメージ** = アプリと部品を固めた「設計図・型」。`--build` で作られます。
- **コンテナ** = イメージから起動した「実際に動いている箱」。homenet は frontend と backend の2箱。
- **ボリューム / バインドマウント** = コンテナの中と、あなたのPCのフォルダを**つなぐ**仕組み。homenet は PC の `./data` フォルダを backend の中の `/data` に**つないで**いるので、コンテナを作り直してもデータは消えず、PC側からファイルを直接編集もできます。
- **docker compose** = 複数の箱の起動を `docker-compose.yml` 1枚にまとめて、一括で立ち上げる道具。

## データの流れ（具体例）

### 例1: 画面を開いて NAS の情報を見る

```
1. ブラウザで http://localhost:8080 を開く
2. nginx が React の画面一式を返す（フロント起動）
3. フロントが起動時に「GET /api/devices」を送る
4. nginx が backend に取り次ぐ → FastAPI が DB（homenet.db）を読んで JSON を返す
5. フロントが受け取った22件を地図と一覧に描く
6. あなたが NAS のノードをクリック → 右パネルが NAS の情報に切り替わる（画面内だけで完結。サーバー通信なし）
```

### 例2: 機器を1台追加する

```
1. 「+ add」→ フォームに入力 → 保存
2. フロントが入力を検査（IP の形式など）
3. 「POST /api/devices」で backend に送る
4. FastAPI が中身を検証し、DB に保存（トランザクション）
5. 成功(201)が返る → フロントが一覧を取り直して画面に反映
```

このとき `data/homenet.db`（SQLite）に保存されます。中身を見る・一括編集するときは、画面、または **export → JSON 編集 → import** を使います。

## どのファイルが何の担当？（地図）

```
homenetwork-viewer/
├── docker-compose.yml      ← 2つの箱の起動設定（入口の8080もここ）
├── data/homenet.db         ← あなたの機器データ（SQLite・実体）。バックアップ対象はこれ
├── backend/                ← バックエンド（FastAPI / Python）
│   └── app/
│       ├── main.py         ←  API の窓口（/api/... の入口）
│       ├── models.py       ←  データの形と検証ルール（IP・MAC など）
│       ├── storage.py      ←  SQLite(homenet.db) の読み書き・マイグレーション
│       └── seed/devices.json ← 初期サンプル（初回起動時に DB へ取り込み）
├── frontend/               ← フロントエンド（React + TypeScript）
│   ├── nginx.conf          ←  画面配布 + /api 取り次ぎの設定
│   └── src/
│       ├── views/          ←  画面（ホーム / 詳細 / 追加・編集）
│       ├── components/     ←  画面の部品（地図・一覧・パネルなど）
│       └── lib/            ←  地図のレイアウト計算など
└── docs/                   ← このドキュメント
```
