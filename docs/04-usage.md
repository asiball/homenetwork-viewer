# 04. 使い方

homenet の画面は **3つ**です。ホーム（地図）→ 詳細 → 編集、という流れで使います。

```
[ ホーム / 地図 ] ──クリック──► [ 詳細 / 1台 ] ──✎ edit──► [ 追加・編集フォーム ]
       ▲                              │                          │
       └──────────── ← map ───────────┘──────── cancel / 保存 ────┘
```

---

## ホーム画面（`/`）

起動して最初に出る、ネットワーク全体の地図です。画面は4エリア:

```
┌─ ヘッダー（ブランド／レイアウト切替／更新／+add） ───────────────────────┐
├──────────┬────────────────────────────────┬────────────────────────┤
│ 機器一覧  │        トポロジーマップ          │   選択中の概要           │
│ (左)     │      (中央・radial / spine)      │   (右)                  │
├──────────┴────────────────────────────────┴────────────────────────┤
└─ フッター（up/down 件数・show offline 切替） ─────────────────────────┘
```

### 中央：トポロジーマップ

機器を地図状に表示します。右上の `gateway` を中心に、各機器がつながって見えます。

- **レイアウト切替**: ヘッダー右の **◎ radial（放射状）/ ─ spine（横バス）** で見た目を切り替え。
  - **radial** … ゲートウェイを中心に同心円。全体の雰囲気をつかむのに向く。
  - **spine** … 横一本の線（バス）に機器がぶら下がる。ネットワーク図らしく、人に説明・配線点検向き。
  - 切り替えると URL が `?layout=spine` のように変わります。その URL をブックマークすれば次回も同じ見た目で開けます。
- **ノードをクリック** … その機器を選択。地図が光り、右パネルが切り替わります。
- **マウスを乗せる（hover）** … 機器名と IP がツールチップで出ます。
- **オフライン機器** … 枠が赤系で、線が点線。ひと目で区別できます。

### 左：機器一覧

カテゴリ（Infra / IoT / Media / Mobile / Computer / Misc）ごとにまとまった一覧。各行は「状態ランプ・機器名・IP の末尾（`.10` など）」。

- 行をクリックで選択（地図・右パネルが連動）。
- 一番下の **「+ add device」** で新規追加フォームへ。
- 画面が狭い時（iPad 縦など）は、ヘッダー左の **≡** ボタンで一覧を開閉できます。

### 右：選択中の概要

選択した機器の要点（identity / network / hardware / notes）。一番下の **「詳細を見る →」** で詳細画面へ移動します。

### キーボード操作

- **↑ / ↓** … 一覧内で選択を上下に移動。
- **Enter** … 選択中の機器の詳細画面へ。

### ヘッダー右の「up X/Y」と更新

- **up X/Y** … 全機器のうちオンラインの数。
- **poll · off / 30s / 5m** … 自動更新の間隔。選ぶと、その間隔でデータを取り直します（インポートで一括更新した場合の反映にも使えます）。
- **⟳ refresh** … いますぐ取り直す。
- **synced HH:MM:SS** … 最後に取り直した時刻。

> 🔰 この「更新」は **データを取り直すだけ**で、実際に機器へ ping を飛ばして調べているわけではありません（自動スキャンは将来の拡張）。今は「台帳を読み直す」操作だと考えてください。

### フッターの「show offline」

オフライン機器を**地図・一覧から隠す/表示する**トグル。普段使っていない機器を消してすっきり見たいときに。

---

## 詳細画面（`/d/:id`）

1台を深掘りする画面。上から:

1. **identity** … カテゴリ・機器名・ホスト名 / IP / MAC、オンライン状態のバッジ。
2. **4つの数値カード** … CPU / メモリ / 帯域 / 稼働時間。データが無ければ `—` や `no agent` と表示（**無い値を勝手に作らない**方針）。
3. **network** … IP/MAC/接続種別/ゲートウェイ/DNS など。配線情報（どのスイッチの何番ポート・ケーブル）も分かる範囲で表示。
4. **hardware** … CPU・アーキ・メモリ・筐体・ファーム。
5. **services / open ports** … 開いているポート一覧（情報がある機器のみ）。
6. **storage / volumes** … ドライブと使用率。
7. **connection（直近7日）** … 接続できていた割合の棒グラフ。
8. **ownership** … メーカー・型番・設置場所・購入日・価格・保証・タグ。
9. **notes** … 自由メモ（複数行）。

- 左上の **← map** でホームへ戻る。
- 右上または identity 右の **✎ edit** で編集フォームへ。
- 左の一覧で別の機器をクリックすると、その機器の詳細に切り替わります。

> 🔰 サンプルでは **NAS** と **Pixel 8** だけ詳細データが充実しています。他の機器は項目が `—` 中心ですが、これは「まだ登録していない」だけ。編集フォームから埋められます。

---

## 追加・編集フォーム（`/add`・`/d/:id/edit`）

機器の情報を入力・更新する画面。項目はセクションに分かれています。

- **identity** … id / 表示名 / ホスト名 / IPv4 / MAC / 種別(type) / グループ / オンライン。
  - **id** は機器を識別する不変の符号（英小文字・数字・ハイフン。例 `nas`）。**追加時のみ**設定でき、表示名から自動提案されます。後から変更はできません。
  - IP は `192.168.1.10` 形式、MAC は `AA:BB:CC:00:0A:11` 形式。形式が違うと、その場で赤いエラーが出ます。
- **placement & link** … 接続種別（有線/Wi‑Fi）と、地図上の配置層（ring）。
  - **ring**: `2 = 末端機器（既定）` / `1 = インフラ（AP・NAS など）` / `0 = ゲートウェイ`。
- **hardware (summary)** … CPU / メモリ / ストレージのざっくり値。
- **ownership** … メーカー・型番・設置場所・購入日・価格・保証・タグ（タグはカンマ区切り）。
- **notes** … 自由メモ。

操作:

- **add device / save changes** … 保存。成功すると右下に通知が出て、その機器の詳細へ移動します。
- **cancel** … 破棄して戻る。
- **delete**（編集時のみ） … その機器を削除（確認ダイアログあり）。

> 🔰 編集で保存しても、CPU使用率グラフやポート一覧などの「詳細データ」は**消えずに保持**されます。フォームで触るのは住所録的な項目（名前・IP・所有情報・メモなど）だけ、と考えてください。
