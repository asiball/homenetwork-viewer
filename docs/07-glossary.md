# 07. 用語集

homenet を読む・使うときに出てくる言葉を、やさしい言い回しでまとめました。五十音ではなく**テーマ別**に並べています。

## ネットワークの言葉

| 用語 | やさしい説明 |
|---|---|
| **LAN** | 家やオフィスの中だけのネットワーク。homenet はこの中だけで使います。 |
| **IP アドレス** | ネットワーク上の「住所」。例 `192.168.1.10`。同じ住所が2つあると混乱するので機器ごとに固有。 |
| **IPv4 / IPv6** | IP アドレスの規格。IPv4 は `192.168.1.10` のような数字4組。IPv6 は新しく長い形式。 |
| **MAC アドレス** | 機器（の通信部品）に最初から付いている「製造番号」のような固有 ID。例 `AA:BB:CC:00:0A:11`。IP は変わってもこれは基本不変。 |
| **OUI（メーカー識別子）** | MAC アドレスの**先頭3組**（例 `AA:BB:CC`）。IEEE という団体がメーカーごとに割り当てており、ここを見れば製造元が分かります。homenet は機器登録時に MAC からメーカー名を自動で**提案**します（外部通信なし・採用するかはあなたが決める）。 |
| **ホスト名 / FQDN** | 機器に付ける名前。例 `nas.home.arpa`。人が覚えやすい「あだ名」。 |
| **サブネット / `/24`** | ネットワークの区画の広さ。`192.168.1.0/24` は「192.168.1.0〜255 の範囲」を指します。 |
| **ゲートウェイ** | 家の中と外（インターネット）の出入口になる機器。たいていルーター。 |
| **DNS** | 「名前 → IP アドレス」を調べてくれる電話帳のような仕組み。 |
| **DHCP** | 機器に IP アドレスを自動で割り当てる仕組み。 |
| **ポート（番号）** | 1つの機器の中の「窓口番号」。同じ住所(IP)でも用途別に窓口が分かれる。例: Web は 80、homenet の入口は 8080。 |
| **ARP / ping** | LAN 内の機器を探す/到達確認する基本手段。homenet の将来の自動収集で使う予定。 |
| **スイッチ / ハブ** | 有線 LAN の機器同士をつなぐ「分岐タップ」。どのポートに何をつないだかを homenet で台帳化できます。 |
| **オンライン / オフライン** | その機器がいま通信できる状態か否か。地図で色分け表示されます。 |

## Web・アプリの言葉

| 用語 | やさしい説明 |
|---|---|
| **ブラウザ** | Web を表示するソフト。Chrome / Safari / Edge / Firefox など。 |
| **Web アプリ** | インストールせずブラウザで使うアプリ。homenet もこれ。 |
| **フロントエンド** | 画面・見た目の部分。あなたが見て触るところ。 |
| **バックエンド** | 画面の裏でデータを保管・処理する部分。 |
| **サーバー** | 要求に応えてデータや画面を返す側のソフト/機械。 |
| **HTTP** | ブラウザとサーバーの会話のルール。「お願い→返事」のやり取り。 |
| **リクエスト / レスポンス** | お願い（リクエスト）と返事（レスポンス）。 |
| **GET / POST / PUT / DELETE** | お願いの種類。取得 / 新規作成 / 置き換え更新 / 削除。 |
| **ステータスコード** | 返事に付く結果番号。`200`/`201`=成功、`404`=見つからない、`409`=重複、`422`=入力エラー、`502`=裏方に取り次げない。 |
| **API** | フロントとバックの「決まった会話の窓口」。例 `/api/devices`。 |
| **JSON** | 文字だけのデータ形式。人にも読める。homenet のデータ保存・通信に使用。 |
| **URL / 住所** | `http://localhost:8080/d/nas` のような場所の表し方。 |
| **localhost** | 「このパソコン自身」を指す住所。 |
| **SPA** | Single Page Application。最初に画面一式を読み込み、以後は一部だけ素早く更新する作りの Web アプリ。 |
| **CORS** | 「別の住所どうしの通信」を安全に制御するブラウザの仕組み。homenet は nginx で窓口を1つにして回避。 |

## 使っている道具（技術）の名前

| 用語 | やさしい説明 |
|---|---|
| **React（リアクト）** | 画面を部品（コンポーネント）で組み立てる、フロントエンド作りの定番道具。 |
| **TypeScript** | JavaScript に「型（データの種類）」のチェックを足した言語。打ち間違いを早く見つけられる。 |
| **Vite（ヴィート）** | フロントの開発・ビルドを高速にする道具。 |
| **nginx（エンジンエックス）** | 画面ファイルを配り、`/api` をバックエンドに取り次ぐ高速なサーバーソフト。 |
| **FastAPI** | Python でAPI（バックエンド）を手早く作れる道具。 |
| **Python** | バックエンドを書いている言語。 |
| **Pydantic** | FastAPI で「入力が正しい形か」を検査する仕組み（IP や MAC の形式チェックなど）。 |
| **uvicorn** | FastAPI のアプリを実際に動かすサーバー。 |

## Docker まわりの言葉

| 用語 | やさしい説明 |
|---|---|
| **Docker** | アプリと必要部品を「箱」に詰めて、どのPCでも同じに動かす仕組み。 |
| **イメージ** | 箱の「設計図・型」。`--build` で作られる。 |
| **コンテナ** | イメージから起動した「動いている箱」。homenet は frontend と backend の2つ。 |
| **docker compose** | 複数のコンテナの起動を1ファイル（`docker-compose.yml`）でまとめて指示する道具。 |
| **ボリューム / バインドマウント** | PC のフォルダとコンテナの中をつなぐ仕組み。homenet は `./data` を backend の `/data` に接続し、データを永続化＆手編集可能にしている。 |
| **ポートの公開（`8080:80`）** | 「PC側の番号 : 箱の中の番号」。PC の 8080 を箱の 80 につなぐ、の意味。 |
| **ヘルスチェック** | コンテナが正常稼働しているかを自動確認する仕組み。 |
| **リバースプロキシ** | 受けた問い合わせを裏のサーバーへ取り次ぐ役。homenet では nginx が担当。 |

## homenet 固有の言葉

| 用語 | やさしい説明 |
|---|---|
| **トポロジーマップ** | 機器のつながりを表した地図。radial（放射状）と tree（配線ツリー）の2表示。 |
| **radial / tree** | 地図の2つのレイアウト。放射状 / 実配線の階層ツリー。 |
| **group（グループ）** | 機器のカテゴリ。Infra / IoT / Media / Mobile / Computer / Misc。 |
| **ring（リング）** | 地図上の配置層。0=ゲートウェイ / 1=インフラ / 2=末端機器。 |
| **detail（詳細データ）** | 詳細画面用の追加情報（ポート・ストレージ・所有情報など）。任意。 |
| **part（構成部品）** | 自作PC等の部品（CPU/GPU/ドライブ等）を1つずつ管理する単位。購入日・価格(円)・保証期限・状態（稼働中/予備/引退/故障気味）を持てます。詳細画面で総額と保証切れが見えます。 |
| **build event（構成変更）** | 「いつ何を増設/換装/交換したか」の履歴。部品の add / remove / replace を日付付きで記録します。 |
| **homenet.db** | 機器データの保存先。homenet の「正データ」（SQLite データベース）。一括編集は export → JSON 編集 → import。 |
| **seed（シード）** | 初回に投入される初期サンプルデータ。 |
| **poll / refresh** | データを取り直す操作。現状は台帳の読み直し（実機スキャンは将来）。 |
